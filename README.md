# Autocomplete Interview Project

## Instructions

The goal of this interview is for you to create a **User Interface** that performs **autocomplete** against a dictionary of words using an **API endpoint** that you'll also write.

The frontend interface can be as simple as an input in the middle of a page. It should:

- Display matches based on user input (similar to how Google search autocomplete works).  
- For example, if the user types `foo`, your app should look up results using your API and display them.  
- When the user selects a result, it should **console.log the chosen item**.

> **Note:** Avoid using external libraries that solve most of the problem for you. The goal is to showcase your creativity, ingenuity, and debugging skills, not just to make something that “works.”

The interview aims to see what you can do in a **fixed amount of time** to deliver the **best user experience** possible.
